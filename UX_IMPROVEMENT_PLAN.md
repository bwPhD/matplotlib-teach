# 用户体验优化方案

## 目标
1. 合并参数百科和交互式调整
2. 设置中文字体支持
3. 优化用户体验，让小白容易学习

## 设计方案

### 1. 界面结构优化

**当前问题**：
- 用户需要在"交互式调整"和"参数百科"之间切换
- 层级较深，需要多次点击

**优化方案**：
- **统一界面**：在交互式调整界面中，每个参数旁边添加"查看选项"按钮
- **侧边栏帮助**：使用 `st.expander` 显示参数说明和常用选项
- **实时预览**：交互式调整和选项预览同时显示
- **简化导航**：减少层级，一目了然

### 2. 具体实现

#### 方案 A：侧边栏帮助面板（推荐）
```
┌─────────────────┬─────────────────────┐
│  参数控制区     │   图表预览区        │
│  - linestyle    │                     │
│    [选择框]     │    [实时图表]       │
│    [查看选项]   │                     │
│                 │                     │
│  - linewidth    │                     │
│    [滑块]       │                     │
│    [查看选项]   │                     │
└─────────────────┴─────────────────────┘
         ↓
┌─────────────────────────────────────┐
│  [展开] 参数说明和选项预览           │
│  - linestyle 说明                   │
│  - 常用选项预览图                   │
│  - 代码示例                         │
└─────────────────────────────────────┘
```

#### 方案 B：内联帮助（更简洁）
```
┌─────────────────────────────────────┐
│  参数控制区（带内联帮助）            │
│                                     │
│  linestyle                          │
│  [选择框: -]  [ℹ️ 查看所有选项]     │
│  ┌─────────────────────────────┐   │
│  │ [展开] 选项预览和说明        │   │
│  │ - 预览图                     │   │
│  │ - 代码示例                   │   │
│  └─────────────────────────────┘   │
│                                     │
│  linewidth                          │
│  [滑块: 2]  [ℹ️ 查看说明]           │
└─────────────────────────────────────┘
```

### 3. 中文字体配置

- 优先使用开源字体：思源黑体、文泉驿等
- 自动检测系统可用字体
- 全局设置，所有图表自动应用

### 4. 用户体验优化

- **简化语言**：使用更通俗易懂的说明
- **视觉引导**：使用图标和颜色区分
- **渐进式披露**：默认显示常用选项，高级选项可展开
- **实时反馈**：参数变化立即反映在图表上
- **代码提示**：显示最小可运行代码
